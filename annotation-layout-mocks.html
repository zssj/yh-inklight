2. Hidden by default, click highlight for popover

cleanest

Harness

The page remains visually quiet. Highlights appear inline, but notes stay hidden until the reader clicks the marked phrase.

This keeps Obsidian close to its normal reading rhythm and avoids permanent side UI.

It is clean, but users lose the margin-note scanning behavior.

3. Always-on right sticky lane

desktop-only variant

Harness

This layout feels like a serious research reader with persistent side commentary always visible.

The right lane becomes the single home for sticky notes, replies, colors, and resolved state.

On narrow windows, this can still collapse into a popover without changing the source Markdown.

Primary noteLine up with the highlighted paragraph and follow scroll.
Reader rhythmAlways visible, so the page feels like a PDF reader.
Compact ruleOnly one sticky lane. No left/right split, no layout drama.

Useful as the desktop half of option 1, but too rigid if it refuses to collapse on narrow screens.

4. Right Leaf only, no inline sticky lane

most stable

Harness

The document keeps native Obsidian layout. Highlights remain visible in text, and all note cards live in a separate overview panel.

This is easiest to implement and least likely to break themes, but it feels less like a PDF reader.
